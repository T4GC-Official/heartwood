# Contributing Code to the T4GC Conservation Coalition 

1. Github [workflow](https://www.kubernetes.dev/docs/guide/github-workflow/): we adopt the k8s github workflow to manage contributions. Follow these steps when you are ready to code. Be sure to replace your user and project name in the commands you run. 

2. Try and write good commit messages. Whether you hand craft them or use a LLM. Here is a baseline

```
Summarize changes in 50 characters 

More details. Word wrap to 80 chars. 
1. Explain the problem
2. Focus on the "why" 
3. Briefly explain the "what" 
4. You pr explains the "how" - you don't need to

Add unfinished items to a bug, eg: 
Bugs: #123
```

3. Try and ask yourself the following questions at all times

* Phase 1: Is the idea behind my contribution sound? 
* Phase 2: Is the contribution architected correctly? 
* Phase 3: Is the contribution polished? 

Phase 1 is the most important, regardless of whether the change was asked or you or self motivated. Discuss phases 1 and 2 with prospective reviewers. 

Regarding phase 3, you will need to squash the nit-picker within you that wants to brush up every single line. Focus on the main things: 

* Robustness
* Security/Privacy
* Documentation

Flag doubts. Don't be silent just because you don't have time to fix it.
